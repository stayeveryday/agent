# Fine-Tuning Concepts

## 1. 微调是什么

微调就是在已有基础模型上，用你自己的样本继续训练，让模型更稳定地学会某种输入输出模式。

它更像 Java 里的：

```text
基类能力已经有了
你通过少量样本覆盖/强化某些行为
```

但它不是给模型安装数据库。

## 2. SFT 是什么

SFT 是 Supervised Fine-Tuning，也就是监督微调。

数据形式通常是：

```text
用户输入 -> 标准答案
```

例如：

```text
用户：帮我查 INC-1001 状态
助手：{"intent":"ticket_query","ticket_id":"INC-1001"}
```

这类任务很适合你的 Agent 项目，因为它能训练模型更稳定地产生结构化意图。

## 3. LoRA 是什么

LoRA 是一种参数高效微调方法。

它不更新整个大模型的大量参数，而是在模型的部分线性层旁边加一小组 adapter 参数，只训练这些 adapter。

直观理解：

```text
Full fine-tuning:
  修改整个模型

LoRA:
  冻结原模型
  只训练很小的外挂 adapter
```

优点：

- 显存占用更低
- 训练更快
- 输出文件更小
- 可以保留基础模型，再切换不同 adapter

## 4. 这次为什么选 LoRA + SFT

因为你的目标是一个月内学会最简单的模型微调。

最小闭环应该是：

```text
小模型 + 小数据 + LoRA + SFT
```

不要一开始碰 RLHF、DPO、大模型全参微调、分布式训练。

## 5. 微调和 RAG 的边界

适合微调：

- 输出格式
- 回答风格
- 意图分类
- 工具参数抽取
- 固定流程话术

适合 RAG：

- 企业知识库
- 制度文档
- 产品手册
- 工单处理规范
- 经常变化的信息
- 需要权限控制和引用来源的信息

一句话：

```text
知识用 RAG，行为模式用微调。
```

## 6. 下午要达成什么水平

下午不要求训练出可上线模型。

下午只要求你能讲清楚并跑通：

```text
数据格式是什么
SFT 在训练什么
LoRA 训练了哪些参数
adapter 输出在哪里
如何加载 adapter 推理
为什么这不替代 RAG
```
