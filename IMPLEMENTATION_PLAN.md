# 企业 RAG Agent 改造计划

## 目标

基于 `LangChain` 搭建一个适用于企业知识问答场景的 Agent，覆盖：

1. Query 改写
2. 意图识别
3. 向量数据召回
4. 工具使用
5. 总结
6. 反思
7. 记忆

## 方案拆解

### 第一阶段：最小可运行版本

目标是先把链路跑通，不先绑定复杂基础设施。

- 编排框架：`LangGraph`
- 模型层：`ChatOpenAI`
- 向量层：`OpenAIEmbeddings + InMemoryVectorStore`
- 记忆层：本地 `JSONL + 向量化召回`
- 工具层：示例服务台工具、时间工具

对应模块：

- `agent.py`: 主流程图编排
- `prompts.py`: 各节点 Prompt
- `schemas.py`: 结构化输出和状态定义
- `vectorstore.py`: 知识库装载与检索
- `tools.py`: 默认工具
- `memory.py`: 记忆持久化与召回

### 第二阶段：企业化增强

把示例能力替换成真实企业组件。

1. 将知识库切片结果接入正式向量库
2. 在检索前增加用户身份、部门、租户、权限过滤
3. 接入真实工具系统，例如工单、OA、CRM、ERP
4. 为反思节点增加低置信度二次检索或人工兜底
5. 将长期记忆迁移到 Redis / Postgres 等持久层

## 节点设计

### 1. Query 改写

- 输入：用户原问题 + 最近记忆
- 输出：适合检索的标准化问题
- 作用：补全代词、省略上下文、统一术语

### 2. 意图识别

- 判断是否需要：
  - 检索知识库
  - 调用工具
  - 直接闲聊响应

### 3. 向量召回

- 根据改写后的问题召回 Top-K 文档
- 后续建议增加 metadata filter

### 4. 工具使用

- 由模型决定是否调用工具
- 当前示例工具：
  - `current_time`
  - `create_helpdesk_ticket`

### 5. 总结

- 汇总检索结果、工具结果、记忆上下文
- 输出面向用户的最终回答

### 6. 反思

- 对答案做质量复核
- 判断是否缺信息、是否需要追问、置信度高低

### 7. 记忆

- 将本轮有效信息压缩成长期记忆
- 用于后续会话上下文补全
